# BabyCare

BabyCare는 **육아 기록, 사진 추억, 가족 공유, AI 요약을 하나로 묶은 스마트 육아 플랫폼**입니다.

- **모바일 앱(Flutter)**: 기록 입력, 실시간 요약 확인, 알림, LLM 대화, IoT 연동
- **웹(React)**: 사진첩 열람, 성장 타임라인, 가족 공유, 리포트 조회
- **서버(FastAPI)**: 인증, 기록 저장, 사진 업로드, LLM 오케스트레이션, 리포트 생성, 권한 관리

## 제품 비전

BabyCare는 부모가 흩어진 육아 정보를 하나의 흐름으로 관리할 수 있도록 돕습니다.

- 앱은 **입력과 실시간 사용**에 집중합니다.
- 웹은 **열람과 공유**에 집중합니다.
- `빠기스`는 기록을 이해하고 요약해 주는 **육아 특화 LLM 에이전트**입니다.

## 핵심 사용자 경험

### 앱 사용자

- 수유, 수면, 배변, 체온, 일정 등을 빠르게 기록
- 자연어/음성으로 기록 입력
  - 예: `빠기스야, 11시 20분에 분유 90ml 먹었어`
- 홈에서 오늘 현황과 요약 확인
- 사진을 성장 이벤트와 함께 저장
- 알림 및 IoT 연동 상태 확인

### 웹 사용자

- 큰 화면에서 사진첩과 성장 타임라인 탐색
- 월별/이벤트별 앨범 열람
- 가족 공유 링크 또는 권한 기반 접근
- 인쇄용/보관용 리포트 확인

## 핵심 서비스 구성

### 1. Flutter 앱

- 육아 기록
  - 수유: 모유/분유/이유식, 시간, 양, 메모
  - 수면: 시작/종료, 자동/수동 입력
  - 배변/기저귀: 소변/대변/혼합, 상태 메모
  - 건강: 체온, 몸무게, 키, 약 복용
  - 일정: 병원 예약, 예방접종, 검사 일정
- 빠기스 LLM
  - 자연어 기록 저장
  - 대화형 질의응답
  - 주간 요약
  - 월령 기반 가이드
  - 이상 징후 참고 문구 생성
  - 감성 메시지 생성
- 사진/추억
  - 사진 업로드
  - 태그/이벤트 분류
  - 대표 사진 지정
  - 첫 뒤집기, 첫 이유식 같은 이벤트 생성
- 알림
  - 수유 주기, 예방접종, 약 복용, 성장 기록 리마인드
- 가족 공유
  - 개별 계정 + 가족 그룹 연결
  - 역할 기반 권한 관리
- IoT 연동
  - 온습도, 공기질, 울음 감지, 수면 환경 지표

### 2. React 웹

- 사진첩
  - 월별 앨범
  - 이벤트별 앨범
  - 태그 필터
  - 슬라이드 보기
  - 다운로드/공유
- 성장 타임라인
  - 사진 + 기록 + 이벤트 통합 뷰
  - 첫 순간 모아보기
  - 월령별 요약
- 리포트 열람
  - 주간 수면/수유 패턴
  - 성장 추이
  - 빠기스 요약 코멘트
- 가족 공유 페이지
  - 초대 링크
  - 권한 있는 가족만 열람
  - 비공개/공개 앨범 구분

### 3. FastAPI 서버

- 인증/인가
- 가족/아기 관리
- 육아 기록 CRUD
- 사진 업로드 및 미디어 관리
- 타임라인/이벤트 구성
- 빠기스 LLM 오케스트레이션
- 요약 리포트 생성
- 알림 스케줄링
- IoT 디바이스 및 센서 데이터 수집

## 빠기스(Paggis) 설계 원칙

빠기스는 단순 챗봇이 아니라 다음 역할을 갖는 육아 도우미입니다.

1. **Record Parser**
   - 자연어를 구조화된 기록으로 변환
2. **Insight Generator**
   - 최근 3일/7일 데이터를 바탕으로 패턴 분석
3. **Memory Companion**
   - 사진/이벤트 기반 추억 코멘트 생성
4. **Care Assistant**
   - 월령/상황 기반 일반 육아 가이드 제공

### 안전 정책

- 의료적 **진단은 제공하지 않음**
- 응급 권고 기준은 참고용으로만 안내
- 위험 단어 감지 시 전문가 상담/병원 문의 권고
- 중요한 문구에는 `정확한 판단은 전문가 상담이 필요합니다` 정책 반영

## MVP 범위

### 앱

- 로그인
- 아기 등록
- 수유/수면/배변 기록
- 사진 업로드
- 빠기스 자연어 기록
- 가족 초대
- 홈 요약

### 웹

- 사진첩 열람
- 타임라인 열람
- 가족 공유 페이지

### 서버

- 인증
- CRUD API
- 파일 업로드
- LLM 파싱/요약
- 알림 스케줄러

> IoT는 MVP 이후 2차 단계로 확장합니다.

## 개발 단계

1. **Phase 1. 코어 구축**
   - 인증, 가족/아기, 육아 기록, 사진 저장, 홈 대시보드
2. **Phase 2. 빠기스 탑재**
   - 자연어 기록 저장, 요약 리포트, 대화형 Q&A
3. **Phase 3. 웹 앨범**
   - 사진첩, 타임라인, 공유 기능
4. **Phase 4. IoT**
   - 센서 연동, 환경 모니터링, 알림 자동화

## 권장 기술 스택

- **Flutter**: Riverpod, GoRouter, Dio, Freezed, Firebase Messaging 또는 OneSignal
- **React**: React + TypeScript + Vite, TanStack Query, Zustand, React Router, Tailwind CSS
- **FastAPI**: SQLAlchemy 2.x, Alembic, PostgreSQL, Redis, Celery/Dramatiq, S3 스토리지, JWT 인증

## 센서/캠 확장

향후 BabyCare는 단순 환경 센서뿐 아니라 **캠 연동**, **울음/소리 감지**, **움직임 감지**를 백엔드 IoT 도메인으로 확장할 수 있도록 설계합니다. 현재 백엔드 스캐폴드는 이 확장을 반영할 수 있게 `devices` API 시작점을 포함합니다.


## DDD 기반 백엔드 시작

백엔드는 처음부터 **DDD 스타일 모듈 구조**를 적용해 `api -> application -> domain` 경계가 흐려지지 않도록 관리합니다. 자세한 내용은 [`backend/docs/ddd.md`](backend/docs/ddd.md)를 참고하면 됩니다.
